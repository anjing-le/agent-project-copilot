# 七大 Agent 实战课品牌与教学物料设计规划

## 1. 整体课程定位

课程主标题建议：

> Agent 工程实战矩阵：一套脚手架，七类业务智能体

这套课程的核心不是反复讲底层技术栈，而是把统一的 `infra-dev-scaffolding` 作为工程底座，展示同一套目录结构、质量规范、前后端契约和交付流程，如何生长出七种不同业务类型的 Agent 项目。

课程面向希望把 Agent 从 Demo 推进到真实工程的人群，包括开发者、技术负责人、AI 产品经理和解决方案顾问。每个项目都要求可运行、可提交、可演示，有明确业务边界、前后端模块、验收脚本和 GitHub 展示材料。

一句话定位：

> 同一个工程脚手架，七种 Agent 业务解法；每一课都是一个完整项目。

核心传播点：

- 不是概念课，而是七个可运行的 Agent 工程。
- 不是零散 Demo，而是统一工程底座上的业务矩阵。
- 不是只学工具调用，而是学习 Agent 在不同业务场景中的边界设计。
- 不是一次性代码演示，而是可持续提交、复盘和扩展的 GitHub 项目集。

## 2. 七大 Agent 课程矩阵

| 项目 | Agent 类型 | 业务主题 | 核心关键词 | 课程交付 |
| --- | --- | --- | --- | --- |
| `agent-aigc` | 创作型 Agent | 全模态创作平台 | 多模型调度、Prompt 优化、图文音视频、异步生成、资产管理 | 创作工作台、任务队列、资产库、生成记录 |
| `agent-knowledge` | 知识型 Agent | RAG 智能知识库 | 文档解析、切片、Embedding、检索、引用、防幻觉 | 知识库管理、检索问答、引用证据、召回评估 |
| `agent-customer-service` | 服务型 Agent | 智能电商客服 | 多轮对话、RAG、规则兜底、会话管理、转人工 | 客服会话台、知识推荐、规则兜底、工单升级 |
| `agent-order-ops` | 执行型 Agent | 订单运营执行 | Tool Calling、审批、幂等、审计、失败补偿 | 订单操作台、工具执行流、审批记录、审计日志 |
| `agent-data-analyst` | 分析型 Agent | 数据分析工作台 | Text-to-SQL、指标口径、图表、洞察、安全查询 | 分析问答、SQL 预览、图表生成、洞察报告 |
| `agent-finance-reconcile` | 审核型 Agent | 财务对账复核 | 票据解析、规则校验、异常复核、审计证据 | 对账工作台、异常池、复核记录、证据链 |
| `agent-project-copilot` | 协作型 Agent | 项目交付协作 | 任务拆解、风险跟踪、会议纪要、周报复盘 | 项目空间、任务拆解、风险雷达、行动项看板 |

矩阵设计重点：

- 七个项目共用同一套工程底座，减少学习者在工程环境上的重复成本。
- 每个项目只突出一种 Agent 主能力，避免课程主题互相稀释。
- 每个项目都保留完整工程交付动作，包括后端 API、前端页面、契约校验、文档和演示路径。
- 七个项目组合后形成一套从生成、检索、服务、执行、分析、审核到协作的业务 Agent 图谱。

## 3. 从入门到进阶的学习路径

### 阶段一：统一工程底座

学习目标：

- 理解 `infra-dev-scaffolding` 的目录结构、脚本、文档习惯和质量规范。
- 掌握前后端契约、OpenAPI、服务边界和探针脚本的作用。
- 建立每个 Agent 项目都要可运行、可提交、可演示的交付意识。

建议产出：

- 本地启动一个标准脚手架项目。
- 完成一次契约检查、构建检查和 Git 提交。
- 明确后续七个项目都从同一工程底座演化。

### 阶段二：Agent 基础能力

推荐学习项目：

1. `agent-aigc`
2. `agent-knowledge`

学习重点：

- 创作型 Agent 解决内容生成和资产沉淀。
- 知识型 Agent 解决检索、引用和可信回答。
- 这两个项目分别对应生成式能力和 RAG 能力，是后续业务 Agent 的基础。

### 阶段三：业务闭环 Agent

推荐学习项目：

1. `agent-customer-service`
2. `agent-order-ops`

学习重点：

- 服务型 Agent 强调会话、规则、人工兜底和客户体验。
- 执行型 Agent 强调工具调用、审批、幂等、失败补偿和审计。
- 学员开始理解 Agent 不只是回答问题，还要嵌入真实业务流程。

### 阶段四：可信决策 Agent

推荐学习项目：

1. `agent-data-analyst`
2. `agent-finance-reconcile`

学习重点：

- 分析型 Agent 强调自然语言分析、指标口径、安全查询和洞察表达。
- 审核型 Agent 强调规则校验、异常识别、人工复核和证据链。
- 学员学习如何让 Agent 的输出可解释、可追溯、可审查。

### 阶段五：团队协作 Agent

推荐学习项目：

1. `agent-project-copilot`

学习重点：

- 把任务拆解、会议纪要、风险雷达、行动项和周报复盘串成协作闭环。
- 理解 Agent 在团队协同中的边界：辅助整理、提醒、追踪和复盘，而不是替代项目负责人。
- 学会把多个业务模块组织成一个交付型工作台。

## 4. 视觉与品牌方向

品牌关键词：

- 工程底座
- 业务生长
- Agent 矩阵
- 可运行项目
- GitHub 作品集

视觉母题：

> 一个统一基座，生长出七个业务 Agent 模块。

建议视觉结构：

- 主视觉中间是统一工程脚手架，周围延展出七张 Agent 项目卡片。
- 每张项目卡片使用统一版式，但有独立识别色和关键词。
- 所有宣传图都保留工程元素，例如目录树、提交记录、接口契约、运行截图和模块图。
- 避免把课程包装成抽象 AI 概念课，视觉上要让人看到真实项目感。

推荐色彩系统：

- 主色：工程蓝或深墨色，用于统一品牌和 GitHub 质感。
- 辅色：七个项目分别使用一枚识别色。
- 背景：浅色适合课程首页和学习路线图，深色适合视频封面和工程实战海报。

七项目识别色建议：

| 项目 | 识别色方向 | 表达 |
| --- | --- | --- |
| `agent-aigc` | 紫红 / 洋红 | 创作、多模态、灵感 |
| `agent-knowledge` | 靛蓝 / 青蓝 | 知识、检索、可信 |
| `agent-customer-service` | 绿色 / 青绿 | 服务、响应、连接 |
| `agent-order-ops` | 橙色 / 琥珀 | 执行、流程、动作 |
| `agent-data-analyst` | 蓝色 / 天蓝 | 分析、图表、洞察 |
| `agent-finance-reconcile` | 金色 / 深绿 | 审核、财务、证据 |
| `agent-project-copilot` | 青色 / 蓝绿 | 协作、进度、交付 |

## 6. 七个单项目物料文案

### 6.1 `agent-aigc` 创作型 Agent

海报标题：

> 创作型 Agent：全模态创作平台

封面文案：

> 让 Agent 管理 Prompt、模型、任务和资产

教学卖点：

- 从单次生成升级到可管理的创作工作流。
- 学习多模型调度、Prompt 优化、异步生成和资产管理。
- 将图文音视频生成任务沉淀成可追踪的工程模块。

课程标签：

- 多模型调度
- Prompt 优化
- 异步生成
- 资产管理
- 创作工作台

### 6.2 `agent-knowledge` 知识型 Agent

海报标题：

> 知识型 Agent：RAG 智能知识库

封面文案：

> 让回答带来源，让知识可追溯

教学卖点：

- 完整走通文档解析、切片、Embedding、检索和引用回答。
- 讲清 RAG 系统如何降低幻觉，而不是只做一个问答框。
- 让学员理解知识库 Agent 的召回、引用和可信边界。

课程标签：

- 文档解析
- 向量检索
- 引用证据
- 防幻觉
- RAG 工作流

### 6.3 `agent-customer-service` 服务型 Agent

海报标题：

> 服务型 Agent：智能电商客服

封面文案：

> 会回答，也知道什么时候该转人工

教学卖点：

- 把多轮对话、商品知识、售后规则和人工兜底放进同一个客服闭环。
- 学习会话状态、知识推荐、规则命中和升级工单。
- 让 Agent 服务真实用户，而不是只完成一次问答。

课程标签：

- 多轮对话
- 规则兜底
- 会话管理
- 转人工
- 客服工作台

### 6.4 `agent-order-ops` 执行型 Agent

海报标题：

> 执行型 Agent：订单运营执行

封面文案：

> 让 Agent 调工具，但每一步都有边界

教学卖点：

- 学习 Tool Calling 如何进入订单查询、改价、取消、补偿等操作流。
- 重点处理审批、幂等、失败补偿和审计记录。
- 让学员理解执行型 Agent 的核心不是能不能调用工具，而是能不能安全地调用工具。

课程标签：

- Tool Calling
- 审批流
- 幂等控制
- 失败补偿
- 操作审计

### 6.5 `agent-data-analyst` 分析型 Agent

海报标题：

> 分析型 Agent：数据分析工作台

封面文案：

> 从自然语言到可信图表和洞察

教学卖点：

- 学习 Text-to-SQL、指标口径、SQL 预览、安全查询和图表生成。
- 将分析过程拆成提问、生成、校验、执行、解释和沉淀。
- 让 Agent 输出的不只是答案，而是可复核的数据分析过程。

课程标签：

- Text-to-SQL
- 指标口径
- 安全查询
- 图表生成
- 洞察报告

### 6.6 `agent-finance-reconcile` 审核型 Agent

海报标题：

> 审核型 Agent：财务对账复核

封面文案：

> 让异常有证据，让复核有依据

教学卖点：

- 围绕票据解析、规则校验、异常识别和人工复核构建高可信 Agent。
- 强调审计证据、规则解释和复核记录，而不是只给出判断结果。
- 适合展示 Agent 在严肃业务中的可追溯设计。

课程标签：

- 票据解析
- 规则校验
- 异常复核
- 审计证据
- 财务对账

### 6.7 `agent-project-copilot` 协作型 Agent

海报标题：

> 协作型 Agent：项目交付协作

封面文案：

> 让任务、会议、风险和周报形成闭环

教学卖点：

- 学习项目空间、任务拆解、会议纪要、风险雷达、行动项和周报复盘。
- 把 Agent 放在团队协作边界内，辅助整理、追踪、提醒和复盘。
- 展示一个从信息收集到行动追踪的项目交付工作台。

课程标签：

- 任务拆解
- 会议纪要
- 风险雷达
- 行动项看板
- 周报复盘

## 7. 总课程大纲与项目拆分

### 总课程大纲

1. 课程总览：一套脚手架如何生长出七类 Agent
2. 工程底座：目录结构、脚本、契约、质量规范和 Git 提交流程
3. 创作型 Agent：从 Prompt 到全模态资产工作流
4. 知识型 Agent：从文档到可信 RAG 问答
5. 服务型 Agent：从会话到客服业务闭环
6. 执行型 Agent：从工具调用到安全执行
7. 分析型 Agent：从自然语言到数据洞察
8. 审核型 Agent：从规则判断到证据链复核
9. 协作型 Agent：从任务拆解到交付复盘
10. 结课复盘：如何基于同一脚手架扩展第八个 Agent

### 单项目标准拆分

每个项目建议拆成 6 节：

1. 业务场景与 Agent 边界
2. 领域模型与模块拆分
3. 后端 API 与契约设计
4. 前端工作台与演示路径
5. 质量脚本、探针和验收
6. README、Demo Evidence 和 GitHub 展示

### 七个项目课程重点

| 项目 | 第 1 节重点 | 第 2 节重点 | 第 3 节重点 | 第 4 节重点 | 第 5 节重点 | 第 6 节重点 |
| --- | --- | --- | --- | --- | --- | --- |
| `agent-aigc` | 创作流程 | 任务与资产模型 | 生成任务 API | 创作工作台 | 异步任务探针 | 作品集展示 |
| `agent-knowledge` | 知识问答边界 | 文档与切片模型 | 检索问答 API | 知识库界面 | 引用验收 | RAG 说明 |
| `agent-customer-service` | 客服边界 | 会话与工单模型 | 对话 API | 客服工作台 | 规则兜底验收 | 服务闭环 |
| `agent-order-ops` | 执行权限边界 | 订单与工具模型 | 工具调用 API | 订单操作台 | 幂等与审计 | 风险说明 |
| `agent-data-analyst` | 分析安全边界 | 指标与查询模型 | 分析 API | 图表工作台 | SQL 安全验收 | 洞察样例 |
| `agent-finance-reconcile` | 审核责任边界 | 单据与异常模型 | 复核 API | 对账工作台 | 证据链验收 | 审核说明 |

## 10. 可复用标语

- 一套脚手架，七种 Agent 业务解法。
- 不是 Demo 集合，是可运行的 Agent 工程矩阵。
- 从同一个工程底座，长出不同业务智能体。
- 每一课一个完整项目，每个项目一条业务闭环。
- 让 Agent 课程从概念演示走向工程交付。
- 学会的不只是 Agent，而是 Agent 如何进入业务系统。
- 用七个 GitHub 项目，建立你的 Agent 工程作品集。
